# PRD — AI 기반 자동화 MVP 빌더

---

## 제품 개요

**한 줄 요약**: 자연어 요구사항 입력 → AI 분석 문서 생성 → 사용자 확인 → 실행 가능한 MVP 코드 자동 생성 및 GitHub 저장소 전달

**존재 이유**: 개인·소규모 팀이 아이디어를 코드로 검증하기까지 걸리는 초기 개발 비용과 시간 병목을 AI 파이프라인으로 제거한다.

**해결하는 문제**:
- 비개발자는 외주 비용과 노코드 플랫폼 종속성 문제에 직면
- 개발자는 보일러플레이트·환경 설정에 반복적으로 시간을 소모
- 기존 AI 빌더는 불투명한 토큰 과금과 기술 스택 제약이 존재

## 핵심 기능 목록

### Must-have (MVP 필수)

| # | 기능명 | 설명 |
|---|--------|------|
| F-01 | 요구사항 입력 | 자연어 + 기술 스택/아키텍처 선호 입력 UI |
| F-02 | AI 분석 문서 생성 | Claude Agent SDK 기반 ERD, API 스펙, 시스템 아키텍처 초안 자동 생성 |
| F-03 | 분석 문서 검토 및 피드백 | 사용자가 생성된 문서를 검토하고 수정 요청 또는 확정 |
| F-04 | 태스크 분해 | 확정된 문서 기반으로 구현 태스크 단위 자동 정의 |
| F-05 | TDD 기반 코드 생성 | 테스트 코드 우선 작성 → 구현 → 리팩터링 방식으로 코드베이스 자동 생성 |
| F-06 | GitHub 저장소 생성 및 코드 업로드 | 사용자 GitHub OAuth 인증 후 신규 저장소 생성 및 생성된 코드 자동 push |
| F-07 | 실행 가능성 보장 | 모든 생성 코드는 `docker compose up --build` 기준으로 즉시 실행 가능 |
| F-08 | 실시간 진행 상황 스트리밍 | AI 생성 단계의 진행 상황을 SSE로 사용자에게 실시간 노출 |
| F-09 | 사용자 API Key 관리 | 사용자 Claude API Key를 암호화 저장 및 파이프라인 실행 시 사용 |
| F-10 | GitHub OAuth 로그인 | GitHub 계정으로 로그인 및 저장소 접근 권한 획득 |

## 사용자 스토리

### 핵심 플로우

| # | 스토리 |
|---|--------|
| US-01 | As a 비개발자 창업자, I want to 자연어로 내 서비스 아이디어를 입력하고, So that AI가 ERD와 API 스펙을 자동 생성해주어 개발자에게 전달할 수 있는 문서를 얻고 싶다. |
| US-02 | As a 개발자, I want to 기술 스택과 아키텍처를 직접 선택하여 코드베이스를 생성하고, So that 나의 선호 스택으로 즉시 개발을 이어갈 수 있다. |
| US-03 | As a 사용자, I want to 생성된 분석 문서를 검토하고 수정 요청을 남긴 뒤 확정하고, So that 내가 원하는 방향으로 코드가 생성되도록 제어할 수 있다. |
| US-04 | As a 사용자, I want to 내 GitHub 저장소에 실행 가능한 코드가 자동으로 업로드되고, So that `docker compose up --build` 명령어 하나로 즉시 서비스를 실행해볼 수 있다. |
| US-05 | As a 사용자, I want to AI 코드 생성 진행 상황을 실시간으로 확인하고, So that 언제 완료될지 예측하고 기다릴 수 있다. |
| US-06 | As a 사용자, I want to 내 Claude API Key를 안전하게 등록하고, So that 플랫폼 과금 없이 실제 사용량만큼만 비용을 지불할 수 있다. |

## 비기능 요구사항

### 성능 목표

| 항목 | 목표 |
|------|------|
| 일반 API 응답 시간 | 2초 이내 (P95) |
| 분석 문서 생성 시간 | 60초 이내 (SSE 스트리밍으로 진행 상황 노출) |
| 코드 생성 시간 | 5분 이내 (SSE 스트리밍으로 진행 상황 노출) |
| 동시 파이프라인 처리 | 최소 10개 동시 세션 지원 |

### 보안 요구사항

- 사용자 Claude API Key: AES-256 암호화 저장 (C-SEC-02)
- GitHub Access Token: 암호화 저장 및 만료 관리 (C-SEC-02)
- 모든 API 인증 필수 (C-SEC-01)
- 생성 코드는 플랫폼 서버에서 직접 실행하지 않음 (C-SEC-03)

### 확장성 고려사항

- **[결정]** AI 코드 생성은 NestJS async/await로 처리한다. Node.js 이벤트 루프 기반이므로 I/O 대기 중 다른 요청이 병렬 처리된다. 작업 큐(BullMQ 등)는 MVP 이후 도입을 고려한다.

### 지원 플랫폼

- 웹 브라우저: Chrome, Firefox, Safari 최신 버전
- 데스크톱 퍼스트 (모바일은 MVP 이후)
- 플랫폼 기술 스택: Next.js 14+ / NestJS / PostgreSQL / Docker
- AI 레이어: Claude Agent SDK (TypeScript), Anthropic Claude 모델
