## 1. 시장 개요

### 시장/카테고리 정의

mvp-builder는 **AI 코드 생성 + 자동 저장소 배포** 카테고리에 속한다. 보다 구체적으로는 "자연어 요구사항만으로 즉시 실행 가능한 프로젝트 코드베이스를 생성하고 GitHub에 자동 업로드하는 서비스"로, 다음 두 영역의 교차점에 위치한다.

- **AI App Builder**: Lovable, Bolt.new, v0 등 — 자연어로 웹 앱을 생성
- **Boilerplate/Scaffolding 자동화**: 프로젝트 초기 설정 자동화 도구

### 시장 트렌드

- **Vibe Coding의 부상**: 2025년 기준 자연어 기반 코드 생성이 개발 워크플로의 표준으로 자리 잡고 있다. 비개발자도 AI를 통해 앱을 직접 만들려는 시도가 급증하고 있다.
- **브라우저 내 개발 환경 경쟁**: Bolt.new(WebContainer), Replit(클라우드 IDE)이 로컬 환경 없이도 앱을 생성하고 실행하는 경험을 제공하며 시장을 확장 중이다.
- **GitHub 연동 강화**: 생성된 결과물을 GitHub로 내보내는 기능이 경쟁사들의 핵심 차별화 요소로 부각되고 있다.
- **크레딧 기반 과금 피로**: 사용량 기반 크레딧 모델의 불투명한 비용 구조에 대한 사용자 불만이 점증하고 있다.

## 2. 경쟁사/대안 목록

### 직접 경쟁 1 — Lovable

- **URL**: https://lovable.dev
- **유형**: 직접 경쟁
- **핵심 기능**:
  - 자연어 프롬프트로 풀스택 웹 앱 생성
  - Supabase 네이티브 연동 (DB 스키마 자동 생성)
  - GitHub 양방향 동기화 (생성 결과물 push/pull)
  - 원클릭 배포
  - 브라우저 내 비주얼 편집기
- **타겟 사용자**: 비개발자, 1인 창업자, 빠른 프로토타이핑이 필요한 소규모 팀
- **가격 모델**:
  - Free: 월 30 메시지 한도, 공개 프로젝트만 가능
  - Pro: $25/월, 월 100 크레딧, 비공개 프로젝트
  - Business: $50/월
- **강점**:
  - 비개발자가 접근하기 가장 쉬운 UX
  - GitHub 동기화가 강력하여 로컬 편집 후 재업로드 가능
  - Supabase 연동으로 DB까지 자동 구성
- **약점/한계**:
  - 크레딧 소진 시 워크플로 단절 (빠른 반복 개발에 불리)
  - 복잡한 비즈니스 로직, 커스텀 API 연동에서 빈번한 오류
  - 생성 결과물이 Lovable 생태계(Supabase, Vercel)에 종속
  - 보안 취약점 이슈 보고됨 (2025년 4월 VibeScamming 취약점)
  - 코드 디버깅 시 직접 편집 어려움

---

### 직접 경쟁 2 — Bolt.new (StackBlitz)

- **URL**: https://bolt.new
- **유형**: 직접 경쟁
- **핵심 기능**:
  - WebContainer 기반 브라우저 내 풀스택 개발 환경
  - 자연어 프롬프트로 앱 생성 및 실시간 실행
  - 터미널, 패키지 매니저, 파일시스템 접근 가능
  - 빌트인 호스팅, 커스텀 도메인, Serverless Functions
  - 생성 중 에러 자동 감지 및 수정
- **타겟 사용자**: 해커톤 참가자, 개발자 (로컬 환경 없이 빠르게 시작하고 싶은)
- **가격 모델**:
  - Free: 월 100만 토큰
  - Pro: $20/월 (1,000만 토큰)
  - Pro 50: $50/월 (2,600만 토큰)
- **강점**:
  - 실시간 코드 실행 확인 (생성과 동시에 앱 동작 확인 가능)
  - 로컬 환경 설치 없이 즉시 시작
  - 파일시스템·터미널 접근 등 개발자 친화적 환경
- **약점/한계**:
  - 토큰 기반 요금제로 비용 예측이 어렵고 디버깅 시 급격히 소모
  - 프로젝트 규모가 커질수록(15~20 컴포넌트 초과 시) 컨텍스트 품질 저하
  - 복잡한 상태 관리, 인증 플로우, 서드파티 통합에서 신뢰성 하락
  - 결과물이 GitHub 저장소로 직접 연결되지 않음 (별도 export 필요)

---

### 직접 경쟁 3 — Replit Agent

- **URL**: https://replit.com/products/agent
- **유형**: 직접 경쟁
- **핵심 기능**:
  - 자연어로 앱 생성 및 클라우드 환경 자동 설정
  - 자체 테스트 루프 (코드 실행 → 오류 감지 → 자동 수정)
  - Agent 3: 최대 200분 연속 자율 실행
  - 데이터베이스, 인증, 외부 서비스 연동 자동 구성
  - 생성 결과 즉시 배포 (Replit 클라우드)
- **타겟 사용자**: 개발자, 스타트업 팀, 기술적 배경이 있는 1인 창업자
- **가격 모델**: Replit Core 플랜 포함 (월 $25 내외, 별도 Agent 사용량 과금)

> 가정: Replit Agent의 정확한 현행 가격 구조는 확인하지 못해 추론으로 작성했습니다.

- **강점**:
  - 가장 강력한 자율 실행 능력 (200분 연속, 자가 테스트)
  - 클라우드 IDE 환경이 있어 생성 후 즉시 수정/실행 가능
  - 전체 인프라 구성까지 자동화 (DB, 인증, 배포)
- **약점/한계**:
  - Replit 클라우드 환경에 종속됨 (로컬 개발 환경으로 전환 복잡)
  - GitHub 저장소로의 자동 업로드 흐름이 mvp-builder와 달리 주요 기능이 아님
  - 비개발자에게는 진입 장벽이 여전히 존재
  - 비용 예측이 어려움

### 직접 경쟁 4 — GitHub Copilot Workspace

- **URL**: https://githubnext.com/projects/copilot-workspace
- **유형**: 직접 경쟁 (부분)
- **핵심 기능**:
  - GitHub 이슈/태스크를 자연어로 설명하면 코드 편집 플랜 자동 생성
  - 기존 저장소 기반 코드 수정 및 PR 생성
  - 단계별 스펙 → 플랜 → 코드 생성 워크플로
  - GitHub 네이티브 통합 (이슈 → PR 흐름)
- **타겟 사용자**: 기존 GitHub 코드베이스가 있는 개발자
- **가격 모델**: GitHub Copilot Pro ($10/월) 이상 유료 플랜에 포함
- **강점**:
  - GitHub 생태계 완전 통합 (이슈, PR, 코드리뷰 연계)
  - 기존 프로젝트 수정에 특화
- **약점/한계**:
  - 신규 저장소를 처음부터 생성하는 기능에 특화되지 않음
  - GPT-4o 기반으로 Anthropic Claude 기반 서비스 대비 코드 품질 차이 가능
  - 비개발자에게는 사용 진입 장벽이 높음
  - 복잡한 세션에서 토큰 소모가 빠름 (300 premium request 한도)

- **URL**: https://v0.dev
- **유형**: 간접 경쟁 (Frontend 특화)
- **핵심 기능**:
  - 자연어/이미지(Figma) 기반 React + Tailwind 컴포넌트 생성
  - Vercel 원클릭 배포
  - 생성 결과 즉시 미리보기
- **타겟 사용자**: 프론트엔드 개발자, UI/UX 디자이너
- **가격 모델**: Free ($5 크레딧), Premium ($20/월, $20 크레딧), Team ($30/유저/월)
- **강점**: 고품질 React UI 생성, Figma 연동, Vercel 생태계 통합
- **약점/한계**: 프론트엔드 코드만 생성 (백엔드 미지원), GitHub 연동 불안정 보고

- **약점/한계**: 여전히 초기 보일러플레이트 설정, CI/CD 구성, 테스트 세팅에 수 시간 소요. 비개발자에게는 불가능

## 4. 차별화 포인트

### 차별화 1 — "GitHub 저장소를 즉시 받는" 경험의 완결성

경쟁사 중 **자연어 입력 → GitHub 저장소 자동 생성 → clone URL 제공**까지 원스톱으로 제공하는 서비스는 많지 않다. Lovable은 가능하지만 크레딧 소진 시 흐름이 끊기고, Bolt.new는 GitHub export가 주요 흐름이 아니다. Replit Agent는 자체 클라우드 환경에 종속된다.

mvp-builder는 **"clone → install → run" 성공률 90% 이상**을 출시 전제 조건으로 두고 있어, 결과물의 즉시 실행 가능성을 명시적으로 보장한다.

- 연결되는 페인포인트: 개발자 페르소나의 "생성 결과물을 로컬에서 바로 쓸 수 있어야 한다"는 요구, 비개발자 페르소나의 "결과물을 어디서 어떻게 받는지 모르겠다"는 혼란

### 차별화 2 — 생성 과정의 투명성 (Transparency-first)

경쟁사 대부분은 "생성 중" 스피너 또는 단순한 로그 출력만 제공한다. mvp-builder는 SSE 기반으로 **현재 파이프라인 단계(분석 → 문서화 → 개발 → 테스트)와 전체 퍼센테이지**를 실시간 노출하는 것을 핵심 원칙(C-VAL-02 Transparency)으로 삼는다.

- 연결되는 페인포인트: 비개발자의 "AI가 지금 뭘 하고 있는지 모르겠다"는 불안감, 3~10분 대기 중 이탈 방지

### 차별화 3 — 기술 스택 직접 지정 옵션 (Progressive Disclosure)

Lovable은 기술 스택 선택지가 없고 Supabase + Vercel에 종속된다. v0은 React + Tailwind만 지원한다. mvp-builder는 **기본 UX는 단순하게 유지하면서, 개발자 옵션(기술 스택, 아키텍처, 배포 방식)을 접어두었다가 펼치는 Progressive Disclosure 패턴**으로 제공한다.

- 연결되는 페인포인트: 시니어 개발자의 "원하는 기술 스택이 따로 있는데 선택지가 없다"는 불만, 프레임워크 종속 우려
