## 1. 화면 목록 (전체 인벤토리)

| ID | 화면명 | 진입 경로 | 주요 목적 |
|----|--------|----------|----------|
| S-01 | 랜딩 페이지 | 최초 접속 (`/`) | 서비스 소개 및 GitHub OAuth 로그인 유도 |
| S-02 | GitHub OAuth 로그인 화면 | S-01 "GitHub으로 시작하기" 클릭 → GitHub 동의 페이지 (external) | GitHub OAuth 인증 동의 (GitHub 페이지) |
| S-03 | 분석 문서 확인 화면 | 생성 진행 화면(S-07) `awaiting_feedback` 이벤트 수신 후 (동일 페이지 내 상태 전환) | 분석 문서(ERD, API 설계, 아키텍처) 확인 + 승인 또는 피드백 제출 |
| S-04 | (제거됨) | — | 이메일 인증 완료 화면. GitHub OAuth 전환으로 불필요. |
| S-05 | (제거됨) | — | 이메일+비밀번호 로그인 화면. GitHub OAuth 전환으로 불필요. |
| S-06 | 메인 생성 페이지 | 로그인 후 (`/`) | 요구사항 입력 + 개발자 옵션 + 생성 시작 (핵심 화면) |
| S-07 | 생성 진행 화면 | S-06 생성 버튼 클릭 후 (동일 페이지 내 상태 전환) | SSE 실시간 진행률 표시, `awaiting_feedback` 시 분석 문서 화면(S-03)으로 전환 |
| S-08 | 생성 완료 화면 | S-07 생성 완료 이벤트 수신 후 (동일 페이지 내 상태 전환) | clone URL 표시 및 복사, 테스트 리포트 표시 |
| S-09 | 생성 이력 페이지 | 네비게이션 "이력" 클릭 (`/history`) | 과거 생성 목록 조회 및 clone URL 재확인 |
| S-10 | 생성 상세 페이지 | S-09 목록 항목 클릭 (`/history/:jobId`) | 특정 생성 작업 상세 정보 및 clone URL 확인 |

> 가정: S-06, S-07, S-03, S-08은 동일한 URL(`/`)에서 상태(state)에 따라 화면이 전환된다. 페이지 이동 없이 단일 화면 흐름으로 구성한다. (C-UX-02 — 핵심 흐름은 단일 페이지 완결)

## 2. 화면별 와이어프레임

│    ┌──────────────────────────────────────────────┐    │
│    │  아이디어를 입력해보세요...                    │    │

### S-02 GitHub OAuth 로그인 (랜딩 페이지 → GitHub 외부 페이지)

> 참고: S-02는 GitHub 자체 OAuth 동의 페이지이므로 mvp-builder UI가 아니다. S-01에서 버튼 클릭 시 GitHub로 리다이렉트된다.

**S-01 랜딩 페이지의 로그인 버튼 영역:**
```
│                   [GitHub으로 시작하기 →]               │
│                (GitHub 계정으로 1초 만에 시작)           │
```

**GitHub OAuth 동의 후 콜백 처리 중 화면 (로딩):**
```
┌─────────────────────────────────────────────────────────┐
│  [로고] mvp-builder                                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              GitHub 계정 연동 중...                      │
│              [로딩 스피너]                               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**OAuth 실패 에러 화면:**
```
┌─────────────────────────────────────────────────────────┐
│  [로고] mvp-builder                                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    ✗ GitHub 로그인에 실패했습니다                        │
│                                                         │
│    GitHub에서 권한을 거부하거나 오류가 발생했습니다.     │

│    └──────────────────────────────────────────────┘    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### S-03 분석 문서 확인 화면 (동일 페이지, `awaiting_feedback` 상태)

> 가정: 생성 파이프라인의 `analyzing` 단계 완료 후 `awaiting_feedback` SSE 이벤트 수신 시 이 화면으로 전환된다. 사용자는 분석 문서를 검토하고 승인 또는 피드백을 제출한다.

```
┌─────────────────────────────────────────────────────────┐
│  [로고] mvp-builder          [이력]  [john-doe ▾] [로그아웃] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    분석이 완료되었습니다. 내용을 확인해주세요.            │
│                                                         │
│    ┌──────────────────────────────────────────────────┐ │
│    │  탭: [ERD]  [API 설계]  [아키텍처]                │ │
│    ├──────────────────────────────────────────────────┤ │
│    │                                                  │ │
│    │  ## ERD                                          │ │
│    │                                                  │ │
│    │  ```mermaid                                      │ │
│    │  erDiagram                                       │ │
│    │    users {                                       │ │
│    │      uuid id PK                                  │ │
│    │      ...                                         │ │
│    │    }                                             │ │
│    │  ```                                             │ │
│    │                                                  │ │
│    └──────────────────────────────────────────────────┘ │
│                                                         │
│    수정이 필요하다면 피드백을 입력해주세요. (선택)       │
│    ┌────────────────────────────────────────────────┐  │
│    │  예: users 테이블에 role 컬럼을 추가해주세요.   │  │
│    │                                     0 / 2,000자 │  │
│    └────────────────────────────────────────────────┘  │
│                                                         │
│    ┌─────────────────────┐  ┌────────────────────────┐  │
│    │   피드백 반영 후 개발   │  │     이대로 개발 시작    │  │
│    └─────────────────────┘  └────────────────────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### S-05 (제거됨) — 이메일+비밀번호 로그인 페이지

> GitHub OAuth 단일 인증으로 전환됨에 따라 별도 로그인 페이지가 불필요해졌다. 로그인은 S-01 랜딩 페이지의 "GitHub으로 시작하기" 버튼을 통해 이루어진다.
