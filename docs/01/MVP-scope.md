# MVP 범위 정의서 — AI 기반 자동화 MVP 빌더

---

## In-scope (MVP에 포함)

| # | 기능명 | 포함 이유 | 설명 |
|---|--------|-----------|------|
| F-01 | 요구사항 입력 | 파이프라인 시작점, 없으면 아무것도 동작하지 않음 | 자연어 텍스트 + 기술 스택/아키텍처 선호 선택 UI |
| F-02 | AI 분석 문서 생성 | 핵심 가치 제안 (ERD, API 스펙, 아키텍처 자동화) | Claude Agent SDK 기반 문서 자동 생성 |
| F-03 | 분석 문서 검토 및 피드백 | 사용자 제어권 보장, 없으면 블랙박스 | 문서 확인 후 수정 요청 또는 확정 |
| F-04 | 태스크 분해 | 코드 생성 품질의 직접적 선행 조건 | 확정 문서 기반 구현 단위 정의 |
| F-05 | TDD 기반 코드 생성 | 실행 가능성 보장의 핵심 | 테스트→구현→리팩터링 사이클로 코드베이스 생성 |
| F-06 | GitHub 저장소 생성 및 코드 업로드 | 소유권 이전의 유일한 경로 | OAuth 인증 → 신규 저장소 → 코드 push |
| F-07 | 실행 가능성 보장 | 프로젝트 핵심 차별점 | `docker compose up --build` 즉시 실행 가능 |
| F-08 | 실시간 진행 상황 스트리밍 | AI 생성 특성상 필수 UX | SSE 기반 단계별 진행 상황 노출 |
| F-09 | 사용자 API Key 관리 | 비용 투명성의 기반 | Claude API Key 암호화 저장 및 사용 |
| F-10 | GitHub OAuth 로그인 | F-06 선행 조건, 사용자 식별 | GitHub 계정 기반 인증 |

## MVP 핵심 가치 명제

| # | 핵심 가설 |
|---|-----------|
| H-01 | **실행 가능성**: 사용자가 자연어로 아이디어를 입력하면, `docker compose up --build` 한 명령으로 즉시 실행되는 MVP를 자동으로 받을 수 있다. |
| H-02 | **소유권**: 생성된 코드는 사용자 GitHub 저장소에 직접 전달되며, 이후 독립적인 개발이 가능하다. |
| H-03 | **비용 투명성**: 플랫폼 자체 과금 없이 사용자의 Claude API Key 사용량만큼만 실제 비용이 발생한다. |

## 출시 전제 조건

| # | 조건 | 검증 방법 |
|---|------|-----------|
| P-01 | 요구사항 입력부터 GitHub 업로드까지 전체 파이프라인이 끊김 없이 동작 | E2E 시나리오 테스트 |
| P-02 | 생성된 코드가 `docker compose up --build` 실행 성공 | Docker 빌드 자동화 검증 |
| P-03 | 사용자 Claude API Key가 암호화되어 저장되고 로그에 노출되지 않음 | 보안 검토 |
| P-04 | GitHub OAuth 로그인 및 저장소 생성 동작 | GitHub API 연동 테스트 |
| P-05 | AI 생성 단계에서 SSE로 실시간 진행 상황이 노출됨 | 클라이언트 수신 테스트 |
| P-06 | 분석 문서 검토 → 수정 요청 → 재생성 → 확정 플로우 동작 | 시나리오 테스트 |
| P-07 | 동시 10개 세션에서 파이프라인이 독립적으로 동작 | 부하 테스트 |
