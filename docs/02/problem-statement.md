# 문제 정의서 — AI 기반 자동화 MVP 빌더

---

## 현재 상황 (As-Is)

개인·소규모 팀이 아이디어를 코드로 검증하려면 두 가지 경로 중 하나를 택해야 한다.

**경로 1: 직접 개발**
- 보일러플레이트, 인증, DB 설계, 환경 설정에 수 일~수 주 소요
- AI 코드 생성 도구(Claude.ai, Copilot)를 써도 전체 맥락 유지가 어려워 조각난 코드만 나옴

**경로 2: AI 빌더 플랫폼 (Lovable, Bolt.new)**
- 플랫폼 자체 토큰/크레딧 과금으로 비용 예측 불가, 월 수십만 원 지출 위험
- 결과물이 플랫폼 내부에 갇히거나, 추출 시 스택이 고정되어 독립 개발 어려움
- 코드 생성 과정이 블랙박스라 원하는 방향으로 제어하기 어려움
- `docker compose up --build` 같은 즉각 실행 가능성 보장 없음

---

## 핵심 문제

> **개발자·비개발자 모두**, 아이디어 검증 단계에서 **실행 가능하고 소유권이 보장된 코드베이스**를 얻기까지, **비용 효율적이고 제어 가능한 방법이 없다.**

| 관점 | 문제 |
|------|------|
| 누가 | 아이디어를 검증하려는 개인·소규모 팀 (개발자/비개발자 혼합) |
| 어떤 상황에서 | 초기 MVP를 빠르게 만들어야 하는 시점 |
| 무엇이 문제인가 | 실행 가능하고 기술 스택이 자유롭고 완전히 소유 가능한 코드를 얻을 수단이 없다 |

---

## 목표 상황 (To-Be)

MVP-Builder를 통해:

1. 자연어로 요구사항을 입력하면 **ERD, API 스펙, 아키텍처 문서가 자동 생성**되고 사용자가 검토·수정·확정할 수 있다
2. 확정된 문서를 기반으로 **TDD 방식의 코드베이스가 자동 생성**된다
3. 생성된 코드는 **사용자의 GitHub 저장소에 직접 전달**되며 `docker compose up --build` 즉시 실행된다
4. 과금은 **사용자의 Claude API Key 사용량만** 발생하며 Anthropic 콘솔에서 직접 확인 가능하다
5. 플랫폼 종속성 없이 **완전한 코드 소유권**이 보장된다

---

## 문제 해결 시 기대 효과

### 사용자 관점
- 아이디어 → 실행 가능한 MVP 확보까지 소요 시간: 수 일~수 주 → **수 시간 이내**
- 초기 개발 비용: 외주 수백만 원 → **Claude API 사용량만 (수천 원 수준)**
- 코드 소유권: 플랫폼 종속 → **완전한 GitHub 소유권**

### 비즈니스 관점
- 기존 AI 빌더 플랫폼의 불투명한 과금 구조에 불만족한 사용자 전환
- "코드 소유권 + Docker 실행 보장"이라는 명확한 차별점으로 기술 지향 사용자 타겟팅
- 사용자 API Key 기반 구조로 플랫폼 자체 AI 비용 부담 없음

## 문제의 크기

> 가정: 국내외 AI 앱 빌더 시장은 2024년 기준 연 30% 이상 성장 중이며, Lovable/Bolt.new가 수만 명 이상의 유료 사용자를 보유한 것으로 추정된다. "코드 소유권과 기술 스택 자유도" 요구는 개발자 커뮤니티(Reddit, HackerNews 등)에서 지속적으로 제기되고 있다.

잠재 사용자 규모:
- 국내 스타트업 창업자·1인 개발자: 수만 명 이상
- 글로벌 기준: 기존 AI 빌더 불만족 사용자 수십만 명 이상으로 추정
