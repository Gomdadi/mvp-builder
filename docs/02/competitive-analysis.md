## 시장 개요

### 시장/카테고리 정의

AI 기반 코드 생성 및 앱 빌더 시장 (AI App Builder / Vibe Coding Platform).
자연어 입력으로 웹 애플리케이션 코드를 자동 생성하는 도구들이 포함된다.

### 현재 트렌드와 방향성

- LLM 성능 향상으로 "프롬프트 → 코드" 품질이 급격히 향상되며 시장 급성장 중
- 기존 노코드/로우코드에서 "AI 코드 생성" 패러다임으로 전환
- 플랫폼 종속성 문제와 높은 과금 구조에 대한 사용자 불만이 증가
- 코드 소유권(Code Ownership)과 기술 스택 자유도 요구가 높아지는 추세

## 경쟁사/대안 목록

### 직접 경쟁사

#### Lovable (lovable.dev)
- **유형**: 직접 경쟁
- **핵심 기능**: 자연어 → React 앱 자동 생성, 실시간 미리보기, GitHub 연동, Supabase 연동
- **타겟 사용자**: 비개발자, 빠른 프로토타입이 필요한 스타트업
- **가격 모델**: 프리미엄 (무료 플랜 제한적, 유료 플랜 $20~$100+/월)
- **강점**: 직관적인 UI, 빠른 결과물, Supabase 기본 연동
- **약점/한계**: 플랫폼 자체 크레딧 과금(비용 예측 어려움), React + Supabase 스택 고정, 생성 코드 복잡해질수록 품질 저하, 독립 개발로 이행 시 정리 필요

#### Bolt.new (bolt.new by StackBlitz)
- **유형**: 직접 경쟁
- **핵심 기능**: 브라우저 내 Node.js 실행 환경, 자연어 → 풀스택 앱 생성, 즉시 미리보기
- **타겟 사용자**: 개발자, 빠른 프로토타이핑
- **가격 모델**: 토큰 기반 과금 (무료 한도 후 유료)
- **강점**: 브라우저에서 즉시 실행, 다양한 프레임워크 지원
- **약점/한계**: 토큰 소진 속도가 빠르고 비용 예측 어려움, StackBlitz 환경 종속, 복잡한 백엔드 구조에 한계

#### GitHub Copilot Workspace
- **유형**: 직접 경쟁 (개발자 대상)
- **핵심 기능**: 이슈 → 코드 변경 계획 → PR 자동 생성
- **타겟 사용자**: 기존 GitHub 사용 개발자
- **가격 모델**: GitHub Copilot 구독 포함 ($10~$39/월)
- **강점**: GitHub 네이티브 통합, 기존 코드베이스 맥락 이해
- **약점/한계**: 신규 프로젝트 생성보다 기존 코드 수정에 특화, 비개발자 진입 장벽 높음

### 간접 경쟁 / 대체 수단

#### 외주 개발
- **유형**: 대체 수단
- **특징**: 요구사항 → 개발자 고용 → 결과물 수령
- **약점**: 높은 비용(최소 수백만 원~), 긴 납기, 커뮤니케이션 비용

#### 노코드 플랫폼 (Bubble, Webflow)
- **유형**: 간접 경쟁
- **약점**: 플랫폼 종속성 심각, 복잡한 로직 구현 한계, 독립 코드 추출 불가

## 차별화 포인트

### 1. 분석 문서 우선 생성 + 사용자 확정 단계
기존 경쟁사들은 "자연어 → 코드 즉시 생성" 방식이라 결과물이 블랙박스다. MVP-Builder는 **ERD, API 스펙, 아키텍처 문서를 먼저 생성하고 사용자가 검토 후 확정**한다. 비개발자 페르소나의 "내가 원하는 대로 만들어지지 않는다"는 페인포인트를 직접 해소한다.

### 2. 완전한 코드 소유권 + Docker 실행 보장
경쟁사는 플랫폼 내부에서 실행되거나 추출 시 정리가 필요하다. MVP-Builder는 **사용자 GitHub 저장소에 `docker compose up --build` 즉시 실행 가능한 코드**를 전달한다. 개발자 페르소나의 "이후 독립 개발이 가능해야 한다"는 요구를 충족한다.

### 3. 비용 투명성 (사용자 API Key)
플랫폼 자체 토큰/크레딧 과금이 없다. 사용자의 Claude API Key를 직접 사용하므로 **Anthropic 콘솔에서 실제 사용량을 직접 확인 가능**하다. 비용 예측이 어렵다는 기존 플랫폼의 최대 불만을 해소한다.
