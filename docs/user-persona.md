## 페르소나 1 — 아이디어를 가진 비개발자 창업 준비생

### 기본 정보

- **이름**: 박지수 (가명)
- **나이**: 29세
- **직업**: 마케터 (스타트업 2년차), 사이드 프로젝트로 서비스 창업 준비 중
- **거주지**: 서울 마포구

### 목표

- 머릿속에 있는 서비스 아이디어를 실제로 동작하는 데모 수준의 앱으로 만들고 싶다.
- 투자자나 팀원 모집 미팅 전에 "이런 느낌의 서비스입니다"라고 보여줄 수 있는 결과물이 필요하다.
- 개발자를 고용하거나 외주를 맡기지 않고 직접 만들어보고 싶다.

### 니즈

- 코딩을 배우지 않고도 서비스 아이디어를 구현하고 싶다.
- 결과물이 실제로 작동하는지 즉시 확인하고 싶다.
- 생성 과정에서 AI가 무엇을 하고 있는지 알고 싶다 (불안감 해소).
- 결과물을 나중에 다시 꺼낼 수 있어야 한다.

### 페인포인트

- **진입 장벽**: Lovable, Bolt.new를 써봤지만 생성된 결과물을 어떻게 실행하는지 몰라 포기한 경험이 있다. "clone URL을 받았는데 그게 뭔지 모르겠어요"
- **대기 불안**: AI가 처리 중일 때 "지금 잘 되고 있는 건지" 알 수 없어 새로고침하거나 탭을 닫게 된다.
- **크레딧 단절**: 경쟁 서비스에서 크레딧이 소진되자 갑자기 작업이 멈춰 당혹감을 느꼈다.
- **플랫폼 종속**: Lovable에서 만든 결과물이 Supabase, Vercel에 묶여 있어 나중에 개발자에게 넘길 때 설명하기 어려웠다.

### 현재 사용 중인 대안

- Lovable.dev (무료 플랜 — 월 30 메시지 한도로 반복 수정 어려움)
- Notion/Figma로 화면 기획서만 그려두고 개발은 못 하는 상태
- ChatGPT로 코드 조각을 받아 직접 붙여보다가 실패

### 기술 친숙도

- **하** — 스마트폰과 Google 서비스는 능숙하게 사용하지만, 터미널·IDE·Git 경험 없음. "npm"이 무엇인지 모름.

### 핵심 시나리오

**시나리오 1 — 첫 번째 MVP 생성**
지수는 "동네 소상공인을 위한 간단한 예약 관리 서비스"를 만들고 싶다. 회원가입 후 텍스트 상자에 한 문단으로 아이디어를 설명하고 생성 버튼을 누른다. 화면에 "요구사항 분석 중 (20%)… 코드 생성 중 (60%)…"이 표시되고, 5분 후 GitHub clone URL을 받는다. clone URL을 개발자 친구에게 카카오톡으로 보낸다.

**시나리오 2 — 생성 이력 재확인**
미팅 1주일 후 다시 접속해 대시보드에서 이전에 생성한 프로젝트의 clone URL을 복사한다.

## 페르소나 2 — 빠른 프로토타이핑이 필요한 주니어 개발자

### 기본 정보

- **이름**: 이태양 (가명)
- **나이**: 25세
- **직업**: 웹 개발자 (취업 준비 중, 부트캠프 수료)
- **거주지**: 경기도 성남시

### 목표

- 포트폴리오에 올릴 사이드 프로젝트를 빠르게 시작하고 싶다.
- 기술 스택 선택과 초기 세팅에 드는 시간을 최소화하고 실제 기능 구현에 집중하고 싶다.
- 완성된 코드베이스를 받아 내가 원하는 방향으로 확장하고 싶다.

### 니즈

- 원하는 기술 스택(Node.js + React 등)으로 설정된 프로젝트를 즉시 받고 싶다.
- 생성된 코드가 실제로 `npm install && npm run dev`로 바로 실행되어야 한다.
- 보일러플레이트(ESLint, Prettier, 테스트 세팅, Docker 등) 설정을 손으로 하고 싶지 않다.
- 결과물의 파일 구조를 미리 확인하고 싶다.

### 페인포인트

- **초기 세팅 지옥**: 새 프로젝트를 시작할 때마다 `create-react-app`, `NestJS CLI`, ESLint 설정, Dockerfile 작성 등에 반나절 이상 소모한다.
- **기술 스택 종속**: Lovable, Bolt.new는 기술 스택을 선택할 수 없어 자신이 배운 스택과 다른 코드를 받게 된다. 포트폴리오에 쓰기 어렵다.
- **토큰 비용 불예측성**: Bolt.new 사용 시 디버깅 과정에서 토큰이 빠르게 소모되어 예상치 못한 비용이 발생했다.
- **결과물 실행 실패**: AI가 생성한 코드가 실제로 실행이 안 되는 경우가 있어 디버깅에 시간을 낭비한다.

### 현재 사용 중인 대안

- `create-next-app`, NestJS CLI 등 공식 CLI 도구로 직접 설정 (시간 소모 큼)
- Cursor + Claude로 코드 조각 생성 (전체 프로젝트 구조 생성은 직접 해야 함)
- GitHub에서 보일러플레이트 저장소를 clone해 사용 (최신 상태 유지 어려움)

### 기술 친숙도

- **상** — Git, GitHub, Node.js, React 기본 사용 가능. 터미널 친숙. Docker는 기초 수준.

### 핵심 시나리오

**시나리오 1 — 기술 스택 지정 후 프로젝트 생성**
태양은 "할 일 목록 관리 앱 (회원 인증 포함)"을 만들려 한다. 개발자 옵션 패널을 펼쳐 Backend: Node.js + NestJS, Frontend: React, DB: PostgreSQL을 선택한다. 생성 완료 후 받은 clone URL로 저장소를 clone하고, `npm install && npm run dev`를 실행해 정상 동작을 확인한다. 이후 비즈니스 로직을 직접 추가한다.

**시나리오 2 — 여러 프로젝트 관리**
면접 준비 기간 동안 3~4개의 서로 다른 프로젝트를 생성해 생성 이력에서 각 저장소 URL을 관리한다.

## 페르소나 3 — 검증 속도가 최우선인 시니어 개발자/테크 리드

### 기본 정보

- **이름**: 김현우 (가명)
- **나이**: 35세
- **직업**: 스타트업 CTO (팀원 5명), 혹은 프리랜서 풀스택 개발자
- **거주지**: 서울 강남구

### 목표

- 고객 또는 투자자에게 보여줄 PoC(Proof of Concept) 또는 내부 도구를 하루 이내에 만들고 싶다.
- 매번 반복되는 보일러플레이트 작업 없이 핵심 기능 검증에만 시간을 쓰고 싶다.
- 생성된 코드를 신뢰하고 기반으로 확장할 수 있어야 한다 (테스트, CI/CD 포함).

### 니즈

- 원하는 기술 스택, 아키텍처, 배포 방식을 정확하게 지정할 수 있어야 한다.
- ESLint, Prettier, Jest, GitHub Actions CI까지 설정된 완성도 높은 코드베이스를 받아야 한다.
- 결과물이 프로덕션 수준의 구조를 따라야 한다 (모듈 분리, 에러 처리 등).

### 페인포인트

- **반복 작업 피로**: 새 프로젝트마다 동일한 설정 작업을 반복하는 것에 지쳐 있다. "내 GitHub에 보일러플레이트 레포가 5개나 있는데 전부 조금씩 다르다"
- **AI 결과물 불신**: Bolt.new, Lovable 등에서 생성된 코드가 패턴이 일관되지 않고 프로덕션에 그대로 쓰기 어려운 수준이었던 경험이 있다.
- **기술 스택 제한**: 기존 AI 생성 서비스가 특정 스택(Next.js, Supabase 등)에 편중되어 있어 팀의 기술 스택과 맞지 않는다.
- **GitHub 연동 부재**: 결과물을 GitHub에 올리는 과정을 직접 해야 하는 서비스가 많아 자동화 가치가 반감된다.

### 현재 사용 중인 대안

- 자체 보일러플레이트 저장소를 template으로 clone해 사용
- Claude Code / Cursor로 직접 파일을 생성하며 구성 (2~3시간 소모)
- 팀 내 이전 프로젝트 저장소를 복사 후 수정

### 기술 친숙도

- **상** — Docker, CI/CD, AWS 기초, TypeScript, NestJS, React 모두 숙련. 프로덕션 배포 경험 다수.

### 핵심 시나리오

**시나리오 1 — PoC 빠른 생성**
현우는 "사내 휴가 신청 관리 시스템 (관리자/직원 역할 분리)"을 PoC로 만들려 한다. 요구사항을 상세하게 입력하고 개발자 옵션에서 NestJS + React + TypeScript strict 모드를 지정한다. 생성된 clone URL을 팀원에게 공유하고 30분 만에 데모 미팅을 시작한다.

**시나리오 2 — 결과물 구조 검토 후 확장**
생성된 저장소를 clone해 파일 구조와 모듈 분리 방식을 확인한다. 기존 패턴과 일관성이 있으면 직접 비즈니스 로직을 추가하기 시작한다.
