## 페르소나 1: 개발자 창업자 "박민준"

| 항목 | 내용 |
|------|------|
| **이름(가명)** | 박민준 |
| **나이** | 29세 |
| **직업** | 스타트업 CTO 겸 개발자 (팀원 2명) |
| **거주지** | 서울 |

### 목표
- 새로운 서비스 아이디어를 빠르게 코드로 검증하고 투자자에게 데모를 보여주고 싶다
- 반복적인 보일러플레이트 작성 없이 핵심 비즈니스 로직 개발에 집중하고 싶다
- 팀 규모가 작아 인프라, 인증, DB 설계까지 혼자 해야 하는 부담을 줄이고 싶다

### 니즈
- 기술 스택을 직접 선택할 수 있어야 한다 (사용 중인 Next.js + NestJS 선호)
- 생성된 코드를 받은 후 직접 수정하고 배포할 수 있어야 한다
- 생성 과정에서 ERD, API 스펙이 어떻게 설계됐는지 확인하고 싶다

### 페인포인트
- Lovable, Bolt.new 사용 시 토큰이 너무 빨리 소진되고 비용 예측이 안 됨
- 생성된 코드가 플랫폼 내부에 갇혀 있거나 추출 후 구조가 엉망이라 독립 개발이 어려움
- 기술 스택이 플랫폼에 의해 강제되어 (React + Supabase 고정) 팀 스택과 맞지 않음

### 현재 사용 중인 대안
- Bolt.new (토큰 과금 불만), 직접 보일러플레이트 작성 (시간 소모)

### 기술 친숙도
- 상 (풀스택 개발 경험 5년, Docker/GitHub 능숙)

### 핵심 시나리오
1. 새 SaaS 아이디어 → MVPBuilder에서 기술 스택(Next.js+NestJS) 선택 후 요구사항 입력 → ERD/API 스펙 검토 및 확정 → 코드 생성 → 자신의 GitHub 저장소에서 `docker compose up --build` 실행 → 즉시 데모 가능

## 페르소나 2: 비개발자 기획자 "이수연"

| 항목 | 내용 |
|------|------|
| **이름(가명)** | 이수연 |
| **나이** | 32세 |
| **직업** | 서비스 기획자 (PM), 전 마케터 |
| **거주지** | 서울 |

### 목표
- 개발자 없이 자신의 서비스 아이디어를 실제로 동작하는 프로토타입으로 만들고 싶다
- 외주 개발 비용(수백만 원)을 지불하지 않고 MVP를 검증하고 싶다
- 어떤 기능이 어떻게 만들어지는지 이해하고 개발자와 소통할 수 있는 문서를 갖고 싶다

### 니즈
- 코드를 몰라도 서비스의 구조(ERD, 기능 목록)를 이해할 수 있는 시각적 문서
- 생성 과정에서 "내가 원하는 대로 되고 있는가"를 확인하고 수정할 수 있는 기회
- 최종 결과물이 실제로 실행되는 것을 눈으로 확인하고 싶다

### 페인포인트
- 노코드 툴(Bubble, Webflow)은 복잡한 로직 구현이 안 되고 플랫폼 의존도가 너무 높음
- Lovable 사용 시 결과물이 마음에 들지 않아도 수정 방법을 모름, 계속 재생성하다 크레딧 소진
- 외주 개발자에게 요구사항을 전달해도 결과물이 다르게 나오는 경험 반복

### 현재 사용 중인 대안
- Lovable (크레딧 소진 문제), 피그마로 화면 기획 후 개발자에게 외주

### 기술 친숙도
- 하 (GitHub 계정은 있으나 코드 읽기 어려움, Docker 미경험)

### 핵심 시나리오
1. 간단한 구독 서비스 아이디어 → MVPBuilder에 자연어로 설명 → 생성된 ERD/기능 목록을 확인하고 "결제 기능은 나중에 추가"라고 피드백 → 확정 → 코드 생성 완료 → GitHub 링크 수령 → 지인 개발자에게 전달하여 후속 개발 진행

## 페르소나 3: 1인 개발자 "김태원"

| 항목 | 내용 |
|------|------|
| **이름(가명)** | 김태원 |
| **나이** | 26세 |
| **직업** | 프리랜서 개발자 / 사이드 프로젝트 운영 |
| **거주지** | 부산 |

### 목표
- 여러 사이드 프로젝트 아이디어를 빠르게 검증하고 수익화 가능성을 테스트하고 싶다
- API 과금이 투명하고 예측 가능해야 한다 (개인 예산 관리)
- 생성된 코드베이스를 자유롭게 수정하고 배포할 수 있어야 한다

### 니즈
- 비용 예측 가능성: 내 API Key를 쓰고 실제 사용량만 과금되어야 함
- 짧은 시간 내에 여러 아이디어를 코드로 전환
- 생성된 코드의 품질이 직접 작성한 것과 유사해야 리팩터링이 가능

### 페인포인트
- 기존 플랫폼의 토큰/크레딧 구조로 한 달에 수십만 원 지출, 정확한 비용 파악 불가
- 생성된 코드가 너무 복잡하거나 레거시 패턴이 많아 수정이 더 어려움

### 현재 사용 중인 대안
- Claude.ai에서 직접 코드 생성 요청 (컨텍스트 유지 어려움), GitHub Copilot

### 기술 친숙도
- 중상 (풀스택 가능, Docker 경험 있음, CI/CD 기초 이해)

### 핵심 시나리오
1. 새 사이드 프로젝트 아이디어 → API Key 등록 (1회) → 요구사항 입력 → 문서 확정 → 코드 생성 → 자신의 GitHub에서 클론 후 즉시 개발 시작 → 비용은 Anthropic 콘솔에서 직접 확인
