## 페르소나 1 — 빠른 프로토타이핑이 필요한 주니어 개발자

### 기본 정보

- **이름**: 이태양 (가명)
- **나이**: 25세
- **직업**: 웹 개발자 (취업 준비 중, 부트캠프 수료)
- **거주지**: 경기도 성남시

### 목표

- 포트폴리오에 올릴 사이드 프로젝트를 빠르게 시작하고 싶다.
- 기술 스택 선택과 초기 세팅에 드는 시간을 최소화하고 실제 기능 구현에 집중하고 싶다.
- 완성된 코드베이스를 받아 내가 원하는 방향으로 확장하고 싶다.

### 니즈

- 원하는 기술 스택(Node.js + React 등)으로 설정된 프로젝트를 즉시 받고 싶다.
- AI가 내 요구사항을 어떻게 해석했는지 먼저 확인하고 싶다.
- 분석 문서에서 방향이 맞으면 바로 개발을 진행하고 싶다.
- 생성된 코드가 실제로 `npm install && npm run dev`로 바로 실행되어야 한다.
- 테스트 통과 여부를 clone 전에 확인하고 싶다.

### 페인포인트

- **초기 세팅 지옥**: 새 프로젝트를 시작할 때마다 `create-react-app`, `NestJS CLI`, ESLint 설정, Dockerfile 작성 등에 반나절 이상 소모한다.
- **기술 스택 종속**: Lovable, Bolt.new는 기술 스택을 선택할 수 없어 자신이 배운 스택과 다른 코드를 받게 된다. 포트폴리오에 쓰기 어렵다.
- **AI 결과물 불신**: AI가 내 요구사항을 어떻게 해석했는지 모른 채 코드가 생성되어, 결과물이 의도와 달라도 알아채기 어렵다.
- **결과물 실행 실패**: AI가 생성한 코드가 실제로 실행이 안 되는 경우가 있어 디버깅에 시간을 낭비한다.

### 현재 사용 중인 대안

- `create-next-app`, NestJS CLI 등 공식 CLI 도구로 직접 설정 (시간 소모 큼)
- Cursor + Claude로 코드 조각 생성 (전체 프로젝트 구조 생성은 직접 해야 함)
- GitHub에서 보일러플레이트 저장소를 clone해 사용 (최신 상태 유지 어려움)

### 기술 친숙도

- **상** — Git, GitHub, Node.js, React 기본 사용 가능. 터미널 친숙. Docker는 기초 수준.

### 핵심 시나리오

**시나리오 1 — 기술 스택 지정 후 프로젝트 생성**
태양은 "할 일 목록 관리 앱 (회원 인증 포함)"을 만들려 한다. GitHub 계정으로 로그인 후 개발자 옵션 패널을 펼쳐 Backend: Node.js + NestJS, Frontend: React, DB: PostgreSQL을 선택한다. 생성 시작 후 analyzing 단계가 완료되면 ERD, API 설계, 아키텍처 요약 문서가 화면에 표시된다. 문서를 검토한 후 "승인" 버튼을 눌러 개발 단계를 시작한다. 완료 후 테스트 12개 전체 통과 리포트와 clone URL을 받고, 저장소를 clone해 `npm install && npm run dev`로 정상 동작을 확인한다. 이후 비즈니스 로직을 직접 추가한다.

**시나리오 2 — 분석 문서에서 방향 수정**
생성된 분석 문서를 보니 ERD에 필요한 테이블이 빠져 있다. "수정 요청" 버튼을 눌러 "users 테이블에 role 컬럼을 추가해주세요"라고 피드백을 제출한다. 피드백이 반영된 코드베이스를 받는다.

**시나리오 3 — 여러 프로젝트 관리**
면접 준비 기간 동안 3~4개의 서로 다른 프로젝트를 생성해 생성 이력에서 각 저장소 URL을 관리한다.

## 페르소나 2 — 검증 속도가 최우선인 시니어 개발자/테크 리드

### 기본 정보

- **이름**: 김현우 (가명)
- **나이**: 35세
- **직업**: 스타트업 CTO (팀원 5명), 혹은 프리랜서 풀스택 개발자
- **거주지**: 서울 강남구

### 목표

- 고객 또는 투자자에게 보여줄 PoC(Proof of Concept) 또는 내부 도구를 하루 이내에 만들고 싶다.
- 매번 반복되는 보일러플레이트 작업 없이 핵심 기능 검증에만 시간을 쓰고 싶다.
- 생성된 코드의 아키텍처 방향을 사전에 확인하고, 내 기준에 맞게 조정할 수 있어야 한다.

### 니즈

- 원하는 기술 스택, 아키텍처, 배포 방식을 정확하게 지정할 수 있어야 한다.
- 분석 문서(ERD, API 설계, 아키텍처)를 코드 생성 전에 검토하고 수정 방향을 잡을 수 있어야 한다.
- ESLint, Prettier, Jest, GitHub Actions CI까지 설정된 완성도 높은 코드베이스를 받아야 한다.
- 테스트 리포트로 코드 품질을 검증할 수 있어야 한다.
- 결과가 내 GitHub 계정에 직접 올라와야 한다.

### 페인포인트

- **반복 작업 피로**: 새 프로젝트마다 동일한 설정 작업을 반복하는 것에 지쳐 있다. "내 GitHub에 보일러플레이트 레포가 5개나 있는데 전부 조금씩 다르다"
- **AI 결과물 불신**: Bolt.new, Lovable 등에서 생성된 코드가 패턴이 일관되지 않고 아키텍처 방향이 예상과 다른 경우가 많았다.
- **기술 스택 제한**: 기존 AI 생성 서비스가 특정 스택(Next.js, Supabase 등)에 편중되어 있어 팀의 기술 스택과 맞지 않는다.
- **운영자 계정 종속**: 생성된 결과물이 서비스 운영자 계정 아래에 있어 팀에 공유하거나 CI/CD를 연결하기 어렵다.

### 현재 사용 중인 대안

- 자체 보일러플레이트 저장소를 template으로 clone해 사용
- Claude Code / Cursor로 직접 파일을 생성하며 구성 (2~3시간 소모)
- 팀 내 이전 프로젝트 저장소를 복사 후 수정

### 기술 친숙도

- **상** — Docker, CI/CD, AWS 기초, TypeScript, NestJS, React 모두 숙련. 프로덕션 배포 경험 다수.

### 핵심 시나리오

**시나리오 1 — PoC 빠른 생성 + 아키텍처 검토**
현우는 "사내 휴가 신청 관리 시스템 (관리자/직원 역할 분리)"을 PoC로 만들려 한다. GitHub 로그인 후 요구사항을 상세하게 입력하고 개발자 옵션에서 NestJS + React + TypeScript strict 모드를 지정한다. analyzing 완료 후 아키텍처 문서를 확인하고 "역할 분리를 RBAC 미들웨어로 구현해달라"는 수정 요청을 피드백으로 제출한다. 피드백이 반영된 코드와 테스트 리포트를 받아 자신의 GitHub 계정에서 clone 후 30분 만에 데모 미팅을 시작한다.

**시나리오 2 — 분석 문서 검토 후 즉시 승인**
요구사항이 단순한 CRUD인 경우, 분석 문서를 확인하고 별도 수정 없이 바로 "승인"을 눌러 개발 단계를 진행한다. 테스트 전체 통과 리포트를 확인하고 clone해서 팀원에게 공유한다.
