서론 — Recap

기존 플랫폼의 한계

  • 플랫폼 자체 토큰·크레딧 기반 과금 — 비용 예측이 어렵고 불투명
  • 브라우저 실행 환경 종속 — 기술 스택 선택의 자유도 제한
  • 결과물이 플랫폼 내부에 머무름 — 완전한 코드 소유권 보장 불가
  • 비개발자: 외주 비용 부담 / 개발자: 보일러플레이트 반복 작업에 시간 소모

MVP Builder의 목표

  • 자연어 입력 → 분석 문서 → 사용자 확인 → 실행 가능한 코드 자동 생성
  • 사용자 GitHub에 직접 Repository 생성 — 즉각적인 후속 개발 가능
  • 사용자 API Key 직접 사용 — 플랫폼 과금 없이 투명한 비용 구조
  • Docker 기반 실행 환경 — 기술 스택·아키텍처 선택권 사용자에게 보장
  • 서버 내부 샌드박스 테스팅으로 코드 검증 — 생성 직후 실제 컨테이너에서 실행해 동작 여부 확인
  • docker compose up --build로 즉시 실행 가능한 MVP 전달
핵심 차별성 테스트 과정
Phase 3 — per-task Sandbox
  • 태스크마다 test 파일 → impl 파일 순서로 생성 (TDD)

  • 크로스 태스크 의존성 검증 — per-task에서 잡지 못한 오류 포착
  • 실패 시 Claude debug loop (read_files → generate_implementation_code, 최대 10회)

핵심 차별성

투명한 비용 구조

  • 사용자 API Key를 직접 사용 — 플랫폼 자체 과금 없음
  • 실제 모델 사용량 기준의 투명한 비용 구조
  • 크레딧 소진 걱정 없이 아이디어 검증 가능

기술 스택 자유도

  • 원하는 스택·아키텍처를 자유롭게 지정 가능
  • Docker 기반 실행 환경 — 로컬 및 클라우드 이식성 확보
  • 플랫폼 내부 환경 종속 없음

완전한 코드 소유권

  • 결과물을 사용자 GitHub에 직접 생성
  • 온전한 코드 소유권 — 즉각적인 후속 개발 가능
  • 플랫폼 이탈 후에도 독립 개발 가능

기술적 챌린지

프롬프트 & 에러 수정

  • 프롬프트 설계 난이도: 실행 가능한 코드를 안정적으로 생성하도록 시스템 프롬프트·툴 description을 반복 튜닝 — 작은 지시 변화가 결과물 품질에 큰 영향
  • 토큰 비용 관리: Phase 3에서 파일별 컨텍스트 주입 시 입력 토큰 급증 — 최소 필요 정보만 주입하는 압축 전략 필요
  • 에러 수정 과정의 어려움: Claude가 실패 로그를 받아 파일 수정 시, 잘못된 파일을 고치거나 수정 범위가 과도하게 넓어지는 문제 발생 — 디버그 루프 프롬프트 정밀 설계 필요
  • 컨텍스트 유지: runAgentLoop에서 대화 히스토리 증가로 응답 품질 저하 — tool_result 크기 제한 및 히스토리 트리밍 설계

비동기 & 루프

  • 비동기 큐 처리: BullMQ worker에서 Phase 1→2→3→4 순서 보장 + SSE 실시간 스트리밍을 Redis pub/sub으로 분리 — 워커 재시작 시 중복 실행 방지
  • 테스트 루프 설계: per-task sandbox(3회) → 종합 sandbox(10회)의 이중 루프 — 실패 시 Claude에게 적절한 에러 컨텍스트를 전달하는 포맷 설계
  • Docker 샌드박스 도입: 서버 실행 환경에서 격리된 테스팅 환경을 구축하기 위해 dockerode-compose로 컨테이너를 직접 띄워 Docker 샌드박스 환경 구축
  • 크로스 태스크 의존성: Phase 3 per-task는 이전 파일 없이 실행돼 import 오류 발생 가능 — Phase 4에서 전체 파일 모아 재검증

제한점 & 향후 발전 방향

현재 제한점

  • 지원 기술 스택 범위가 한정적 — 검증된 스택 외 안정성 보장 어려움
  • 복잡한 비즈니스 로직의 생성 코드 품질 보장 어려움
  • Docker 이미지 pull 시간으로 인한 파이프라인 전체 지연
  • 생성 코드의 보안 취약점 자동 검사 미비

향후 발전 방향

  • 더 많은 기술 스택 프리셋 지원 (Spring Boot, Django, Ruby on Rails 등)
  • 코드 품질 자동 리뷰 단계 추가 (정적 분석·보안 스캔)
  • Docker 이미지 캐싱으로 파이프라인 속도 개선
  • 사용자 실시간 피드백 루프 강화 — 생성 중 수정 요청 가능
