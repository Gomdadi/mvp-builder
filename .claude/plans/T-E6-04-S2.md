# T-E6-04-S2: Phase 3 샌드박스 테스트 실행 및 재생성 루프

## Context

현재 Phase3Service.runBackend()는 test + impl 파일을 생성한 뒤 바로 S3에 업로드한다.
실제로 테스트가 통과하는지 검증하지 않아 동작하지 않는 코드가 S3에 올라갈 수 있다.

목표: 생성된 코드를 Docker 환경에서 실제 실행해 통과하면 S3 업로드, 실패하면
에러 로그를 Claude에 주입해 impl을 재생성하는 루프를 추가한다. (최대 3회)

### 최종 결정 사항
- **Docker 실행**: dockerode-compose + dockerode — Claude가 생성한 docker-compose.yml을 dockerode-compose로 실행. 컨테이너 생명주기는 dockerode가 관리
- **환경 파일**: Phase 2가 첫 번째 태스크로 보일러플레이트 태스크 생성 → Phase 3 태스크 1에서 docker-compose.yml + 프로젝트 환경 파일 생성 → S3 저장
- **파일 전달**: 환경 파일 + test/impl 코드를 `/tmp/{uuid}/` 임시 디렉토리에 저장 → docker-compose.yml의 volumes로 컨테이너에 마운트
- **재시도 대상**: impl만 재생성. test 코드는 명세이므로 고정
- **범위**: unit test (mock 기반)만 지원. integration test(DB 연결)는 향후 별도 태스크
- **기술스택 유연성**: Claude가 기술스택에 맞는 docker-compose.yml을 직접 생성 (node:20-alpine, maven:3.9-openjdk-17, python:3.11-slim 등)

### 2단계: dockerode-compose 설치 및 DockerSandboxService 재작성
→ 검증: docker-sandbox.service.spec.ts unit test 통과

**패키지 추가** (`apps/backend/package.json`)
```
dockerode-compose, @types/dockerode (이미 추가됨)
```

**`src/docker/docker-sandbox.service.ts`** 재작성

핵심 메서드:
```typescript
async runTest(
  envFiles: GeneratedFile[],   // S3에서 받은 환경 파일들 (docker-compose.yml 포함)
  codeFiles: GeneratedFile[],  // test + impl 파일
): Promise<{ passed: boolean; output: string }>
```

내부 흐름:
1. `/tmp/{uuid}/` 임시 디렉토리 생성
2. envFiles + codeFiles를 임시 디렉토리에 파일로 저장
3. `new DockerodeCompose(docker, '/tmp/{uuid}/docker-compose.yml', uuid)` 생성
4. `compose.pull()` — image 확보 (없으면 다운로드)
5. `compose.up()` — 컨테이너 실행 (dockerode가 생명주기 관리)
6. 컨테이너 로그 수집 + exit code 확인 (dockerode API)
7. `compose.down()` + `/tmp/{uuid}/` 삭제 — finally에서 항상 정리

**docker-compose.yml 예시 (Claude가 생성)**
```yaml
services:
  test:
    image: node:20-alpine
    working_dir: /app
    command: sh -c "npm ci && npx jest --no-coverage --forceExit"
    volumes:
      - .:/app
```

**`src/docker/docker.module.ts`** 수정
- DOCKER_INSTANCE 토큰으로 dockerode 인스턴스 provide (기존 유지)
- DockerSandboxService provide/export (기존 유지)

**Phase3Service.runBackend() 흐름:**
```
runAgentLoop → test + impl 생성
    ↓
S3에서 _env/ 파일들 다운로드 (docker-compose.yml 포함)
    ↓
DockerSandboxService.runTest(envFiles, [testFile, implFile])
    ↓ 성공
uploadAndComplete() → DONE
    ↓ 실패 (최대 MAX_RETRIES = 3)
에러 로그 + 기존 test 코드 → runWithTool(generate_implementation_code)
impl 파일 교체 후 runTest() 재실행
    ↓ 3회 초과
TaskStatus.FAILED
```
