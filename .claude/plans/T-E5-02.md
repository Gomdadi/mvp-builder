# T-E5-02 SSE 이벤트 페이로드 보강 플랜

## Context

클라이언트가 SSE 이벤트를 받은 후 분석 문서/태스크 목록/파일을 조회하려면
이벤트 페이로드에 관련 id가 포함되어야 한다.
현재 `phase_completed PHASE_1/PHASE_2` 이벤트에 id가 없고,
`task_started/task_completed`에 `taskName`이 없다.

---

## 변경 파일

### 1. `src/sse/sse.types.ts`

`SseEvent`에 두 필드 추가:
```typescript
analysisDocumentId?: string; // phase_completed PHASE_1 시
pipelineRunId?: string;      // phase_completed PHASE_2 시
```
(`taskName`은 이미 있음)

---

### 2. `src/pipeline/pipeline.worker.ts`

**`handleStart` / `handleFeedback`**

`phase1Service.run()`이 `Promise<string>` (analysisDocumentId) 를 반환한다 (`phase1.service.ts:210`).
현재 반환값을 버리고 있으므로 캡처해서 이벤트에 포함:

```typescript
// 변경 전
await this.phase1Service.run(projectId, undefined, session?.claudeApiKey);
await this.sseService.publish(projectId, this.event('phase_completed', { phase: PipelinePhase.PHASE_1 }));

// 변경 후
const analysisDocumentId = await this.phase1Service.run(projectId, undefined, session?.claudeApiKey);
await this.sseService.publish(projectId, this.event('phase_completed', { phase: PipelinePhase.PHASE_1, analysisDocumentId }));
```

**`handleConfirm`**

1. `phase_completed PHASE_2` 이벤트에 `pipelineRunId` 추가:
```typescript
// 변경 전
this.event('phase_completed', { phase: PipelinePhase.PHASE_2 })

// 변경 후
this.event('phase_completed', { phase: PipelinePhase.PHASE_2, pipelineRunId })
```

2. 태스크 enqueue 시 `taskName`을 job data에 포함 (`task.worker.ts`가 꺼내 쓸 수 있도록):
```typescript
// 변경 전
{ projectId, pipelineRunId, taskId: task.id, sessionId }

// 변경 후
{ projectId, pipelineRunId, taskId: task.id, taskName: task.name, sessionId }
```

---

### 3. `src/pipeline/task.worker.ts`

`handleRun` job.data에서 `taskName` 추가 추출 후 이벤트에 포함:

```typescript
// 변경 전
const { projectId, pipelineRunId, taskId, sessionId } = job.data

// 변경 후
const { projectId, pipelineRunId, taskId, taskName, sessionId } = job.data

// task_started / task_completed 이벤트에 taskName 추가
{ type: 'task_started', taskId, taskName, timestamp: ... }
{ type: 'task_completed', taskId, taskName, timestamp: ... }
```

---

## 테스트 수정

- `pipeline.worker.spec.ts`: `phase1Service.run` mock이 `analysisDocumentId` 문자열 반환하도록 수정, `phase_completed PHASE_1` 이벤트에 포함 확인
- `task.worker.spec.ts`: job.data에 `taskName` 추가, `task_started/completed` 이벤트에 포함 확인
